# Lazyweb Reference Log: Malaysia RSSニュース要約 UI改善

Lazyweb MCPツール検索では専用ツールが見つからなかったため、`docs/lazyweb-reference-policy.md` に沿って通常Web調査の観察メモだけを残す。参照元の画像、スクショ、ロゴ、固有UI、文言はコピーせず、日次ダッシュボードとしての抽象パターンだけを利用する。

対象:

- `news/malaysia/index.html`
- 主題: マレーシア国内ニュースの日次RSS要約一覧を、今日読むべき要点がすぐ分かるダッシュボードにする。
- UI方針: ニュースポータル化ではなく、個人用の静的ニュース要約として更新感、カテゴリ件数、上位要点、日別アーカイブを読みやすく整理する。

## 共通パターンまとめ

- 情報階層: 最新日の判断材料を最上部に置き、日別比較と過去アーカイブを分ける。
- 更新感: 生成日時と最新Markdownへのリンクをヘッダーと今日のサマリーで確認できるようにする。
- スキャン性: カテゴリ件数は小さなピルでまとめ、過去日は表に近い行構造で日付、件数、失敗状態を揃える。
- モバイル: 見出し、リンク、件数、本文が縦に詰まりすぎないようにし、主要リンクは押しやすい高さにする。
- 表現: 個人用RSS要約なので、ポータルサイト的な大きなナビ、広告枠、画像カード、トレンド装飾は使わない。

## Safety Review

- 参照元をコピーしていない: はい。URLと抽象的な観察だけを記録した。
- 抽象パターンのみ利用している: はい。情報階層、更新感、一覧密度だけを利用した。
- スクショ原本をGit管理していない: はい。画像やスクショは保存していない。
- ロゴ、人物写真、独自UI、独自イラストを真似していない: はい。
- 認証情報、個人用設定値、MCP接続設定の実体を書いていない: はい。
