# Lazyweb Reference Policy

この文書は、Lazyweb MCPを導入する前に決めておく参照リサーチの運用ルールです。今回の用途は、Webサイト、記事ヘッダー画像、UI改善の参考調査と抽象パターン抽出に限定します。

## 基本方針

- Lazywebは参考調査と抽象パターン抽出のみに使う。
- 参照元をコピーしない。
- 抽象パターンのみ利用する。
- 最終成果物は、参照元とは別のオリジナル表現として作成する。

## Git管理しないもの

- スクショ原本はGit管理しない。
- 調査中の下書き画像、生成途中画像、ローカル作業ファイルはGit管理しない。
- 認証情報、個人用設定値、MCP接続設定の実体はリポジトリにもMarkdown文書にも書かない。

## 真似しないもの

- ロゴ
- 人物写真
- 独自UI
- 独自イラスト
- ブランド固有の配色、構図、装飾、アイコン表現
- 記事ヘッダーやLPの完成ビジュアルそのもの

## 記録するもの

調査で残すのは、素材の複製ではなく観察結果です。記録対象は以下に限定します。

- URL
- 取得日
- 媒体種別
- 抽出パターン
- 使わない要素
- オリジナル案に反映する抽象的な方向性

## 使ってよい抽象パターンの例

- 余白の取り方
- 文字量のバランス
- 情報の階層
- 色数やコントラストの傾向
- 左右分割、中央配置、カード配置などの大まかな構図
- 調査、比較、選定、実験ログなどのテーマ表現

## 禁止する使い方

- 参照元の画像をそのまま公開用素材に使う。
- 参照元のスクショ原本をGitに追加する。
- ロゴ、人物写真、独自UI、独自イラストを再現する。
- 参照元と混同されるほど近い構図、配色、装飾に寄せる。
- 認証情報、個人用設定値、MCP接続設定の実体を文書やコミットに含める。
