# Phase 2E.8: Optional Groq Artifact Multi-Day Review

Date: 2026-06-02
Status: Protocol memo created.

## Summary

Phase 2E.8 is documentation/protocol only.

This phase defines a 3 manual observation days review protocol for optional Groq artifacts generated by the Phase 2E.7 production-adjacent workflow.

Groq production adoption remains deferred.

RSS-rendered Markdown remains the production output.

Groq output must not be committed to news/malaysia/.

This phase does not change workflows, scripts, config, Pages output, Groq prompts, body enrichment behavior, source selection, or `news/malaysia/`.

## Review Window

Review window: 3 manual observation days.

Collection is manual only. No automation is added in this phase.

The review should use artifacts downloaded from GitHub Actions runs of `.github/workflows/malaysia-rss-summary.yml`.

## Primary Run Configuration

Use this primary run configuration for each observation day:

```text
enable_body_enrichment=true
enable_groq_rendering=true
groq_model=llama
force_all_groq=false
debug_groq=true
```

This keeps the production RSS output unchanged while allowing body enrichment plus Groq rendering to be reviewed through artifacts.

## Daily Review Fields

Record these fields for each observation day:

- date;
- run id;
- downloaded artifact path;
- selected item count;
- enriched item count, if body enrichment is enabled;
- body policy counts;
- body fetched count;
- body excerpt used count;
- Groq requested / accepted / fallback counts;
- Groq input path;
- Groq rendering status;
- dateline leakage check;
- guard/debug snippets;
- RSS vs Groq readability notes;
- RSS vs Groq factual-safety notes;
- whether Groq output stayed out of `news/malaysia/`.

## Review Criteria

The review is evidence-gathering only. It does not define production adoption success.

For each day, the artifact review should confirm:

- optional Groq rendering succeeds or fails open;
- RSS-only daily output is not blocked;
- Groq output remains artifact-only;
- dateline leakage is not observed;
- improved summaries do not introduce factual drift;
- Japanese readability is meaningfully better than, or at least not worse than, RSS-rendered Markdown.

## Decision Boundary

Groq production adoption remains deferred.

RSS-rendered Markdown remains the production output.

Groq output must not be committed to news/malaysia/.

A later explicit phase is required before any Groq-rendered Markdown can replace or modify production output.
